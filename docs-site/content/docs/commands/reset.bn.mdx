`/reset` হলো ক্লিন-স্লেট বোতাম। কখনো কখনো আপনি নতুন করে শুরু করতে চান — একটি নতুন ক্যারিয়ার
দিকনির্দেশনা, একটি তাজা প্রোফাইল, অথবা আপনি রিপোটি অন্য কাউকে হস্তান্তর করছেন। `/reset`
**আপনার ডেটা** সরিয়ে ফেলে আর সেই **ফ্রেমওয়ার্কটি** যত্নসহকারে সংরক্ষণ করে যা CareerForge-কে
কাজ করায়, তাই টুলটি পরেও নিখুঁতভাবে চলে।

## কী মুছবেন তা বেছে নিন

| আপনি টাইপ করেন | যা মুছে |
|----------|--------|
| `/reset profile` | আপনার প্রোফাইল স্কিল ফাইলগুলো — খালি টেমপ্লেটে ফিরে যায় |
| `/reset documents` | `documents/`-এ আপনার ফাইলগুলো (ফোল্ডার এবং README থেকে যায়) |
| `/reset all` | উভয়ই |

আপনি যদি কোনো স্কোপ না দেন, এটি আপনাকে বেছে নিতে বলে এবং প্রতিটি কী প্রভাবিত করে তা ব্যাখ্যা করে।

## এটি প্রথমে আপনাকে সবকিছু দেখায়

কিছু মুছে ফেলার আগে, `/reset` একটি **inventory** প্রিন্ট করে:

```text
you:    /reset profile
claude: Profile reset — here's what I'll clear vs. keep:

        Will clear:
          01-candidate-profile.md     (has your data)
          02-behavioral-profile.md    (has your data)
          07 STAR examples            (3 examples)
        Already empty:
          05 profile statements       (template)
        NEVER touched (framework):
          03-writing-style.md, 04 scoring framework,
          06 cover-letter templates, 07 framework + roleplay

        This cannot be undone — your git history is the only recovery.
        Type RESET to proceed.
you:    RESET
claude: Cleared 3 files. Run /setup to rebuild your profile.
```

## নিরাপত্তা রেলিং

- **মুছে ফেলার আগে ইনভেন্টরি** — কী যায় এবং কী থাকে তা আপনি সবসময় ঠিকঠাক দেখেন।
- **আপনাকে অবশ্যই `RESET` টাইপ করতে হবে** বড় হাতের অক্ষরে। অন্য যে কোনো উত্তর "nothing was
  changed" বলে বাতিল করে দেয়।
- **ফ্রেমওয়ার্ক কখনো স্পর্শ করা হয় না** — রাইটিং-স্টাইল গাইড, স্কোরিং ফ্রেমওয়ার্ক,
  কভার-লেটার টেমপ্লেট, এবং ইন্টারভিউ-প্রেপ ফ্রেমওয়ার্ক (আপনার রোলপ্লে গাইডলাইন সহ) সবই টিকে
  থাকে। কেবল আপনার *ডেটা* মুছে ফেলা হয়।
- **সৎ রিপোর্টিং** — এটি তালিকাভুক্ত করে আসলে কী মুছে ফেলা হয়েছিল বনাম কী ইতিমধ্যে খালি ছিল,
  এবং কখনো এমন কিছু মুছেছে বলে দাবি করে না যা সেখানে ছিল না।
- **কোনো undo নেই।** এটি শুরুতেই তা বলে দেয় এবং আপনাকে আপনার git history-র দিকে নির্দেশ করে।

## একটি রিসেটের পরে

আপনি যা মুছেছেন তার জন্য `/reset` স্বাভাবিক পরবর্তী পদক্ষেপটি আপনাকে বলে — প্রায় সবসময়
আপনার প্রোফাইল পুনর্নির্মাণের জন্য [`/setup`](/bn/docs/commands/setup) (আপনি যদি ডকুমেন্ট
মুছে থাকেন তবে প্রথমে সেগুলো যোগ করুন)।

> যেহেতু এটি ধ্বংসাত্মক, `/reset` ইচ্ছাকৃতভাবে ড্যাশবোর্ড Console থেকে **পাওয়া যায় না** — এটি
> কেবল এখানেই চলে, যেখানে টাইপ করা `RESET` নিশ্চিতকরণ আপনাকে রক্ষা করে।
