`/expand` هو عالم الآثار الخاص بملفك الشخصي. لقد أنجزت أكثر مما يقوله ملفك — دورة هنا،
مشروع جانبي هناك، أداة أتقنتها أثناء العمل. يبحث `/expand` عن تلك الكفاءات عبر كل ما
يمكنه الوصول إليه و**يضيفها** إلى ملفك الشخصي. إنه لا يعدّل أو يحذف أبدًا ما هو موجود
بالفعل، لذا فإن تشغيله آمن دائمًا.

## أين يبحث

1. **مستنداتك** — `documents/cv` و`linkedin` و`diplomas` و`references`.
2. **حساب GitHub الخاص بك** — كل مستودع عام، مثبّت أو لا: يقرأ ملفات README ويلاحظ
   اللغات والمواضيع.
3. **روابطك الأخرى** — معرض الأعمال، Kaggle، Google Scholar، موقع شخصي.

## كيف يحوّل ما يعثر عليه إلى مهارة

لكل شيء يكتشفه، يبحث في الويب بطريقتين في آنٍ واحد:

- **البحث المباشر** — المنهج الفعلي لدورة محددة، المهارات الكامنة وراء شهادة ما، وثائق أداة.
- **الاستنتاج** — ما الأساليب وسلاسل الأدوات التي يتطلبها عادةً هذا النوع من العمل.

تُوسَم العناصر المستنتجة دائمًا بأنها مستنتجة، حتى تتمكن من التمييز بين مهارة موثّقة وأخرى
مستنبطة بالمنطق.

## ترى كل شيء قبل كتابة أي شيء

```text
you:    /expand
claude: Read your profile. Scanning documents/, GitHub (14 repos), and your Kaggle…

        Competency Map (new items only)
        ── Technical — Primary ────────────────────────────────────
        PyTorch          GitHub: vision-experiments        direct
        ── Methods & Practices ────────────────────────────────────
        MLflow tracking  Coursera: MLOps Specialisation    both
        ── Domain Knowledge ───────────────────────────────────────
        Time-series forecasting   GitHub: demand-forecaster inference

        Add these? (all / review each / skip / skip a group)
you:    all
claude: Added 7 competencies across 2 files. Report below.
```

خياراتك هي **all** أو **review** (المرور مجموعةً مجموعةً) أو **skip** أو تخطّي فئة
واحدة فقط. لا شيء يُكتب حتى تأذن بذلك.

## إضافي وعديم الأثر التكراري — بحكم التصميم

- **إنه يضيف فقط.** لا تُغيَّر الأسطر الموجودة أو تُزال أبدًا.
- **كل إضافة تحمل ملاحظة مصدر**، مثل *(Coursera — Deep Learning Specialisation)* أو
  *(GitHub — repo: demand-forecaster)*. تلك الملاحظة هي أيضًا الطريقة التي يبقى بها
  `/expand` عديم الأثر التكراري: شغّله مرة أخرى الشهر المقبل وسيجلب ما هو جديد فقط.
- **لا شيء يُختلق.** كل كفاءة تعود إلى مصدر حقيقي أو تُوسَم كاستنتاج، وأي مستند لا يستطيع
  قراءته يُبلَّغ عنه بدلًا من التخمين.

## ماذا تحصل عليه في النهاية

**تقرير توسعة**: ما أُضيف إلى كل ملف (مع المصادر)، وكل مصدر فحصه وعدد الكفاءات التي أنتجها
كلٌّ منها، وأي شيء تخطّاه ولماذا، وأي عناصر تستحق نظرة يدوية. خطوة تالية جيدة بعد ذلك هي
[`/upskill`](/ar/docs/commands/upskill) — أعِد فحص فجواتك الآن بعد أن أصبحت مجموعة
مهاراتك أغنى.

يمكن أيضًا تشغيل `/expand` من وحدة تحكّم لوحة المعلومات؛ فهو يشغّل الأمر نفسه.
