`/reset` साफ़-स्लेट बटन है। कभी-कभी आप दोबारा शुरू करना चाहते हैं — एक नई करियर
दिशा, एक नई प्रोफ़ाइल, या आप रिपॉज़िटरी किसी और को सौंप रहे हैं। `/reset` **आपका डेटा** हटा देता है
जबकि उस **फ़्रेमवर्क** को सावधानीपूर्वक सुरक्षित रखता है जो CareerForge को काम करने योग्य बनाता है, ताकि
उपकरण बाद में भी बिल्कुल ठीक चलता रहे।

## चुनें कि क्या मिटाना है

| आप टाइप करते हैं | मिटाता है |
|----------|--------|
| `/reset profile` | आपकी प्रोफ़ाइल स्किल फ़ाइलें — वापस खाली टेम्पलेट्स पर |
| `/reset documents` | `documents/` में आपकी फ़ाइलें (फ़ोल्डर और README बने रहते हैं) |
| `/reset all` | दोनों |

अगर आप कोई scope नहीं देते, तो यह आपसे चुनने को कहता है और समझाता है कि हर एक किसे प्रभावित करता है।

## यह पहले आपको सब कुछ दिखाता है

कुछ भी मिटाने से पहले, `/reset` एक **इन्वेंट्री** प्रिंट करता है:

```text
you:    /reset profile
claude: Profile reset — here's what I'll clear vs. keep:

        Will clear:
          01-candidate-profile.md     (has your data)
          02-behavioral-profile.md    (has your data)
          07 STAR examples            (3 examples)
        Already empty:
          05 profile statements       (template)
        NEVER touched (framework):
          03-writing-style.md, 04 scoring framework,
          06 cover-letter templates, 07 framework + roleplay

        This cannot be undone — your git history is the only recovery.
        Type RESET to proceed.
you:    RESET
claude: Cleared 3 files. Run /setup to rebuild your profile.
```

## सुरक्षा-रेलिंग

- **मिटाने से पहले इन्वेंट्री** — आप हमेशा ठीक-ठीक देखते हैं कि क्या जाता है और क्या रहता है।
- **आपको `RESET` टाइप करना ही होगा** बड़े अक्षरों में। कोई भी दूसरा जवाब "nothing was
  changed" के साथ रद्द कर देता है।
- **फ़्रेमवर्क को कभी नहीं छुआ जाता** — राइटिंग-स्टाइल गाइड, स्कोरिंग फ़्रेमवर्क,
  कवर-लेटर टेम्पलेट्स, और इंटरव्यू-प्रेप फ़्रेमवर्क (आपके रोलप्ले
  दिशानिर्देशों सहित) सब बचे रहते हैं। केवल आपका *डेटा* मिटाया जाता है।
- **ईमानदार रिपोर्टिंग** — यह सूचीबद्ध करता है कि वास्तव में क्या मिटाया गया बनाम क्या पहले से खाली था,
  और कभी यह दावा नहीं करता कि उसने कुछ ऐसा मिटा दिया जो वहाँ था ही नहीं।
- **कोई undo नहीं।** यह शुरुआत में ही बता देता है और आपको आपके git इतिहास की ओर इंगित करता है।

## reset के बाद

`/reset` आपको बताता है कि जो आपने मिटाया उसके लिए स्वाभाविक अगला कदम क्या है — लगभग हमेशा
[`/setup`](/hi/docs/commands/setup) आपकी प्रोफ़ाइल फिर से बनाने के लिए (अगर आपने दस्तावेज़ मिटाए तो
पहले उन्हें जोड़ें)।

> चूँकि यह विनाशकारी है, `/reset` जानबूझकर डैशबोर्ड Console से **उपलब्ध नहीं** है
> — यह केवल यहीं चलता है, जहाँ टाइप किया गया `RESET` कन्फ़र्मेशन आपकी रक्षा करता है।
