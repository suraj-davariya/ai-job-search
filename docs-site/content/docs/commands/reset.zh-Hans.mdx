title: /reset

---

`/reset` 是“一键归零”按钮。有时你想从头开始——一个全新的职业方向、一份崭新的档案，或者你要把这个仓库交给别人。`/reset` 会移除**你的数据**，同时小心地保留让 CareerForge 正常运转的**框架**，因此之后工具仍能完美运行。

## 选择要清除的内容

| 你输入 | 清除内容 |
|----------|--------|
| `/reset profile` | 你的档案技能文件——恢复为空白模板 |
| `/reset documents` | 你在 `documents/` 中的文件（文件夹和 README 保留） |
| `/reset all` | 两者都清除 |

如果你不传入范围，它会让你选择，并解释每个选项分别影响什么。

## 它会先把全部内容展示给你

在删除任何内容之前，`/reset` 会打印一份**清单**：

```text
you:    /reset profile
claude: Profile reset — here's what I'll clear vs. keep:

        Will clear:
          01-candidate-profile.md     (has your data)
          02-behavioral-profile.md    (has your data)
          07 STAR examples            (3 examples)
        Already empty:
          05 profile statements       (template)
        NEVER touched (framework):
          03-writing-style.md, 04 scoring framework,
          06 cover-letter templates, 07 framework + roleplay

        This cannot be undone — your git history is the only recovery.
        Type RESET to proceed.
you:    RESET
claude: Cleared 3 files. Run /setup to rebuild your profile.
```

## 安全护栏

- **删除前先列清单** —— 你始终能清楚看到哪些会被删除、哪些会保留。
- **你必须输入大写的 `RESET`**。任何其他回复都会以“nothing was changed”取消操作。
- **框架永远不会被触碰** —— 写作风格指南、评分框架、求职信模板，以及面试准备框架（包括你的角色扮演指引）全部保留。只有你的*数据*会被清除。
- **如实报告** —— 它会列出实际清除了什么、哪些原本就已为空，绝不会声称抹掉了某个本就不存在的东西。
- **无法撤销。** 它会预先说明这一点，并提示你查看 git 历史。

## 重置之后

`/reset` 会根据你清除的内容告诉你顺理成章的下一步——几乎总是 [`/setup`](/zh-Hans/docs/commands/setup)，用来重建你的档案（如果你清除了文档，请先添加文档）。

> 由于具有破坏性，`/reset` 被有意设计为**不**在仪表盘控制台中提供——它只在这里运行，由你手动输入的 `RESET` 确认来保护你。
