title: /apply

---

`/apply` は CareerForge の中核です。求人情報を渡すだけで — リンクでも貼り付けた
テキストでも、どちらでも同じように機能します — 7 段階のパイプラインが実行され、
印刷可能な 2 つの PDF と、応募トラッカーへの新しい行で締めくくられます。

```text
/apply https://careers.acme.com/jobs/senior-data-engineer-42
```

## まず、求人が本物かを確認します

詐欺、ゴースト求人、情報収集目的の偽装はどこにでもあり、洗練された掲載内容だけ
では何も判断できません。そのため、何かを起草する前に、`/apply` はその求人が本物
で関わっても安全かを評価し、明快な判定を示します。

| 判定 | 意味 |
|---------|---------------|
| **Verified** | 各シグナルが一致 — 実在する企業、本物らしい職務、立ち止まるほどの危険信号なし。 |
| **Caution** | 何かが少し不自然 — あいまいな雇用主、妙な連絡経路、出来すぎた給与。再確認の価値あり。 |
| **Suspicious** | 複数の危険信号 — 前払い金、個人情報の収集、企業と一致しないドメイン。 |

各判定には、実際に見つかった内容を引用した **判定理由** が添えられます。ラベルを
鵜呑みにするのではなく、あなた自身が判断できるようにするためです。

これは **適合スコアとは意図的に分離** されています。詐欺がスキル面で完璧に一致す
ることもあり、もし正当性が 0〜100 の数値に組み込まれていたら、優れた一致が警告を
平均で打ち消し、見えなくしてしまう恐れがあります。*これは良い適合か?* と
*この求人は本物か?* という二つの問いには、二つの答えがふさわしいのです。

システムは警告し、**判断するのはあなた** です。求人を自動でブロックすることは
決してありません。**Suspicious** 判定の場合は、起草に労力を費やす前に明示的な
確認を求めます — 壁ではなく、減速帯です。

## 次に、適合度を採点して尋ねます

一語も書く前に、5 つの観点からあなたのプロフィールに照らして職務を評価します。

| 観点 | 重み |
|-----------|--------|
| 技術スキルの一致 | 30% |
| キャリアの整合性 | 30% |
| 経験の一致 | 25% |
| 行動・カルチャーフィット | 15% |
| 勤務地と実務面 | 合否判定 |

重み付け合計が判定に対応します。**Strong**（75 以上）、**Good**（60〜74）、
**Moderate**（45〜59）、**Weak**（30〜44）、**Poor**（30 未満）。一つだけ上書きが
あります。勤務地があなたの条件を満たさない場合、スキルの一致がどれほど良くても
判定は Poor になります — 就けない仕事は適合ではありません。

そしていったん止まり、こう尋ねます。**起草を進めますか?** そのゲートはあなたの
ものです。No と言えば何も書かれません。評価だけでも、その実行の価値は十分にあり
ます。

## そして、厳格なルールの中で書きます

- **CV** は求人に合わせて調整され、その言語とページ数はアクティブな
  [ロケールパック](/ja/docs/languages) に従います — それが標準ならレターサイズ
  1 ページ、そうでなければ A4 で 2 ページ。
- **カバーレター** は *求人の言語* で書かれます — デンマーク語の求人にはデンマーク
  語のレター — ちょうど 1 ページで、CV を並べ直すのではなく、あなたが解決できる
  課題に焦点を当てます。
- **すべての主張は面接テストを乗り越えなければなりません。** 面接で気兼ねなく
  裏付けられない一文は、そこにあるべきではありません。誇張はあなたに提示されます。
  *残す、和らげる、削る?*

## 二つ目の AI が一つ目をチェックします

デフォルトでは、新鮮な視点を持つ別のレビュアーが両方の草稿を批評します — 抜けた
キーワード、ありきたりな言い回し、触れる価値のある企業に関する切り口など。メイン
エージェントはその後、良い提案を反映し、企業に関する事実の主張をレターに入れる前
に独立して再検証します。

深さ（とコスト）はあなたが制御します。

| 入力 | 起こること |
|----------|--------------|
| `/apply <posting>` | 企業調査を含むフルレビュー（デフォルト） |
| `/apply --review=quick <posting>` | 企業調査なしのレビュー — より高速 |
| `/apply --review=none <posting>` | レビュアーをスキップ — 最速 |

## 最後に、コンパイル、検査、記録

ドキュメントは PDF にコンパイルされ、*目視で検査* されます — ページ数、孤立した
見出し、フォントの不一致 — そしてきれいになるまで再修正されます。その後、応募は
[トラッカーファイル](/ja/docs/your-data) にステータス **Draft**、適合スコア、両方
の PDF へのパスとともに記録されます。最終検証チェックリスト（事実の正確性、
ターゲティング、一貫性、レイアウト）が一度実行され、行われた調整判断の短い要約が
得られます。

## ロボットも読めるエクスポート

美しい LaTeX の PDF は相手側の人間のためのものですが、最初の読者はしばしばレイア
ウトではなくテキストを解析する **応募者追跡システム（ATS）** です。そこで
`/apply` は、PDF（依然として主要で洗練された出力）と並んで、*同じ* 内容から
機械にやさしい 2 つのバージョンを生成します。

- プレーンテキストの **`.txt`** — ATS が最も確実に取り込めるもの。
- Word の **`.docx`** — 編集可能なドキュメントを求めるポータル向け。

その後 **パースのセルフチェック** を実行します。PDF からテキストを再び抽出し、
あなたの名前、セクション見出し、主要なスキルがすべて復元できることを確認します。
パーサーがつまずくなら、採用担当者のソフトウェアより先にあなたが気づけます。

これらの追加機能は、ツールがインストールされていないときに優雅に劣化します。
`.txt` は常に機能します。`.docx` には [`pandoc`](https://pandoc.org/) が必要で、
パースチェックには `poppler`（`pdftotext`）が必要です。どちらかが欠けていても、
そのステップを注記とともにスキップするだけです — あなたの PDF がそれらで止まる
ことはありません。

## すべての主張を、その出典まで追跡

CareerForge の最も固い約束は、経験を決して捏造しないことです。`/apply` はそれを
*監査可能* にします。**プロベナンス台帳** を出力し、CV のすべての主張を、それを
裏付けるプロフィールファイルにマッピングします。あなたの本物のプロフィールに
追跡できなかったものは **あなたのレビュー用にフラグ付け** され、ひそかに出荷される
ことはありません。

同じ台帳は後で [ダッシュボードのプロベナンスパネル](/ja/docs/dashboard/provenance)
で、主張ごとに、裏付けのないものをハイライトした状態で読めます。

## やらないこと

- **応募の送信。** PDF を確認し、自分で送るのはあなたです。
- **経験の捏造。** 並べ替えや強調は許されますが、持っていないスキルを主張すること
  は許されません。
- **履歴の変更。** `/apply` はトラッカーに行を *追加* するだけで、既存の行を変更
  することは決してありません。

**次に起こること:** あなたの応募は [ダッシュボード](/ja/docs/dashboard) に表示され、
状況の進展に応じてステータスを更新していきます。
