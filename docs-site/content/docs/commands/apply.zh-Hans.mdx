`/apply` 是 CareerForge 的核心。给它一份职位信息——一个链接或粘贴的文本，两者
同样好用——它会运行一条七阶段的流水线，最终产出两份可直接打印的 PDF，以及你申请
追踪表里的一新行。

```text
/apply https://careers.acme.com/jobs/senior-data-engineer-42
```

## 首先，它检查这个职位是不是真的

骗局、幽灵职位和数据收割的幌子到处都是，而一个精美的招聘启事本身什么都说明不了。
所以在起草任何东西之前，`/apply` 会评估这个职位是否真实、是否值得安全地去接触，
并给你一个明明白白的裁定：

| 裁定 | 它意味着什么 |
|---------|---------------|
| **Verified** | 各种信号都对得上——真实的公司、看起来真实的职位、没有值得驻足的危险信号。 |
| **Caution** | 有点不太对劲——含糊的雇主、奇怪的渠道、好到不真实的薪资。值得再看一眼。 |
| **Suspicious** | 多个危险信号——预付费用、个人数据收割、一个与公司不匹配的域名。 |

每个裁定都会附上**背后的理由**，引用自它实际查到的内容，这样你可以自己判断，而
不是去信任一个标签。

这一项是刻意与**匹配分保持分离**的。一个骗局也可能是技能上的完美匹配，而如果把
真实性折叠进那个 0–100 的数字里，一个绝佳匹配就可能把警告平均到看不见。两个
问题——*这是个好的匹配吗？*和*这个职位是真的吗？*——值得两个答案。

系统给出警告；**你来决定**。它绝不会自动屏蔽一个职位。一个 **Suspicious** 的
裁定确实会在它愿意花力气起草之前，要求你明确确认——是一个减速带，而非一堵墙。

## 然后它评估匹配度——并询问

在写下一个字之前，它会从五个维度，对照你的档案评估这个职位：

| 维度 | 权重 |
|-----------|--------|
| 技术技能匹配 | 30% |
| 职业契合 | 30% |
| 经历匹配 | 25% |
| 行为 / 文化契合 | 15% |
| 地点与后勤 | 通过 / 不通过 |

加权总分对应到一个裁定：**Strong**（75+）、**Good**（60–74）、**Moderate**
（45–59）、**Weak**（30–44）、**Poor**（低于 30）。有一个一票否决：如果地点
未通过你的约束，那么无论技能匹配多么好，裁定都是 Poor——一份你接不了的工作，
谈不上匹配。

然后它会停下来询问：**继续起草吗？** 那道关卡是你的。说不，就什么也不会写；
单单那份评估往往就值回这次运行。

## 然后它来写——在严格的规则之内

- **简历**针对职位量身定制，其语言和页数遵循当前生效的[地区包](/zh-Hans/docs/languages)——
  在以单页为常规的地方做成 Letter 尺寸的单页，在不是这样的地方做成 A4 的两页。
- **求职信**用*职位所用的语言*来写——一份丹麦语的职位会得到一封丹麦语的信——恰好
  一页，聚焦于你将解决的问题，而非把你的简历重新罗列一遍。
- **每一条声明都必须经得起面试测试。** 如果你无法在面试中坦然支撑某句话，它就
  不该在那里。牵强之处会标记给你：*保留、弱化，还是删除？*

## 第二个 AI 检查第一个

默认情况下，一位拥有全新视角的独立审阅者会对两份草稿挑刺——遗漏的关键词、套话式
的措辞、关于公司值得一提的角度。然后主 agent 采纳那些好建议，并在任何关于公司的
事实性声明被允许进入你的信件之前，独立地重新核实它。

你来控制深度（以及成本）：

| 你输入 | 会发生什么 |
|----------|--------------|
| `/apply <posting>` | 完整审阅，包括公司研究（默认） |
| `/apply --review=quick <posting>` | 审阅但不做公司研究——更快 |
| `/apply --review=none <posting>` | 跳过审阅者——最快 |

## 最后：编译、检查、记录

文档被编译成 PDF 并接受*视觉检查*——页数、孤立的标题、字体不匹配——然后反复修复，
直至干净。接着这份申请会以状态 **Draft**、匹配分以及两份 PDF 的路径，记录在
[你的追踪文件](/zh-Hans/docs/your-data)里。一份最终核验清单（事实准确性、针对性、
一致性、排版）会运行一次，然后你会得到一份关于它所做定制决策的简短摘要。

## 也有机器人能读的导出

一份精美的 LaTeX PDF 是给那一端的人看的——但第一位读者往往是一个解析文本、而非
排版的**申请人追踪系统（ATS）**。所以在 PDF（仍是主要的、打磨好的输出）旁边，
`/apply` 会用*相同*的内容生成两个机器友好的版本：

- 一份纯文本 **`.txt`** —— ATS 能摄取的最可靠的东西。
- 一份 Word **`.docx`** —— 给那些要求可编辑文档的门户。

然后它会运行一次**解析自检**：它把文本从 PDF 中重新拉出来，确认你的名字、各节
标题和关键技能都能被还原。如果解析器会卡住，你会在招聘方的软件卡住之前就知道。

当这些额外功能所需的工具没有安装时，它们会优雅降级。`.txt` 始终可用。`.docx`
需要 [`pandoc`](https://pandoc.org/)；解析检查需要 `poppler`（`pdftotext`）。
缺少其中任何一个，只是带着一条说明跳过那一步——你的 PDF 绝不会因它们而受阻。

## 每一条声明，都追溯到来源

CareerForge 最坚定的承诺，是它绝不会编造经历。`/apply` 让这件事变得*可审计*：
它会输出一份**来源台账**——你简历上的每一条声明都映射回支撑它的档案文件。任何
它无法追溯到你真实档案的东西，都会**标记出来交由你审阅**，绝不会悄悄发出。

你稍后可以在[仪表盘的来源面板](/zh-Hans/docs/dashboard/provenance)里读到同一份
台账，一条声明一条声明地看，无依据的那些会被高亮。

## 它不会做的事

- **提交申请。** 你来审阅 PDF 并自己发送。
- **编造经历。** 允许重新排序和侧重；声称你没有的技能则不行。
- **触碰你的历史。** `/apply` 只会*添加*一行到你的追踪表——它绝不会修改已有的行。

**接下来会发生什么：** 你的申请会出现在[仪表盘](/zh-Hans/docs/dashboard)里，你
将在那里随着事态发展更新它的状态。
