一次求职会产生你所拥有的一些最私人的数据：你在哪里工作过、你挣多少、你愿意搬去
哪里、哪些公司拒绝了你。CareerForge 的立场很简单——**这些数据只与你有关，与谁
都无关**，所以它绝不会离开你的电脑。

## 你的数据存放在哪里

所有数据，都在本地：

| 数据 | 存放于 | 是否上传过？ |
|------|----------|----------------|
| 候选人档案 | 你的 CareerForge 文件夹 | 否 |
| 生成的简历和求职信 | `cv/`、`cover_letters/` | 否 |
| 申请追踪表 | `job_search_tracker.csv` | 否 |
| 已见职位登记表 | `job_scraper/` | 否 |
| 源文档（简历、文凭……） | `documents/` | 否 |
| 薪资数据（可选） | `salary_data.json` | 否 |

这里没有账户、没有注册、没有遥测、没有分析，也没有什么“匿名使用统计”。就连
这个文档网站，本身也被构建成不抓取任何东西——字体和素材都是内置打包的，而非
从第三方服务器加载。

## 哪些东西确实会走网络——以及在什么时候

诚实要求给出完整的图景。有两样东西会接触互联网，且都只在你主动发起时：

1. **AI 本身。** CareerForge 运行在 Claude Code 上，所以你对话的文本——包括 AI
   正在处理的档案内容——会由 Anthropic 的服务按照[他们的隐私
   条款](https://www.anthropic.com/privacy)进行处理。这和你把 Claude 用于其他
   任何事情是一样的安排；如果那对你的聊天来说可以接受，这里也是同样的安排。
2. **你触发的网络搜索。** `/search` 会查询公开的招聘网站，而 `/apply` 会抓取
   职位信息并核实关于公司的说法。这些看起来就像普通的网页浏览。

而绝不会发生的是：后台上传、你的文件被作为文件发送到任何地方，或者任何
CareerForge 服务器——根本就没有这样一个服务器。

## Git 与那个真正重要的习惯

CareerForge 自带一个 `.gitignore`，它排除了上面每一项个人产物，所以这个文件夹的
历史默认就不会沾上你的数据。

要保持一个习惯，两个字：**私有 fork。** 如果你把自己这份 CareerForge 推送到
GitHub，请把仓库设为私有。你的档案与代码在同一棵文件夹树里，虽然忽略规则会保护
你，但一个私有仓库意味着即便出了差错，东西也仍然只属于你。

## 删除一切

没有账户要注销，也没有支持工单。删掉这个文件夹，CareerForge——连同你求职的每
一丝痕迹——就都没了。这就是“本地优先”的含义。
