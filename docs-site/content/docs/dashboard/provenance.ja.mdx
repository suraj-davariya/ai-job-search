CareerForge の最も古い約束は、経歴を決して捏造しないことです。
**来歴パネル（Provenance panel）** は、その約束を信頼に委ねるのではなく、応募ごとに
*確認* できる場所です。

これは応募の詳細ドロワー内にあります。[応募一覧
テーブル](/ja/docs/dashboard/applications-table)で行を開くと、来歴パネルが
レコードの隣に表示されます。テーラリングされたCVが行うすべての記述を並べ、それぞれの
隣に、それを裏付ける **プロフィール ソース** を示します。

## 何を表示するか

各行は記述とその出典を対にして示します:

- **記述** — スキル、成果、職責など、生成されたCVに表示されるとおりに。
- **その出典** — その記述が引き出されたプロフィール ファイル。ドキュメントが
  *なぜ* それを記載してよいのかが分かります。
- **出典がない場合のフラグ。** `/apply` があなたの実際のプロフィールまで遡れなかった
  記述は、隠されるのではなく **レビュー対象としてハイライト** されます。それこそが
  要点です: 裏付けのない記述は、確認すべきものであり、黙って送り出すものではありません。

## ダッシュボードの他の部分と同様、鏡である

このパネルは **読み取り専用** です。あなたのCV、プロフィール、または
トラッカーを編集することはありません — `/apply` がすでに生成した監査を反映するだけです。
応募のドキュメントと並んで書き込まれたプレーンなファイルから読み取ります:

```text
documents/applications/<company>_<role>/provenance.json
```

これもまた[file-as-DB](/ja/docs/your-data)です: 台帳はあなたのマシン上の
人間が読めるファイルであり、ドキュメントが生成されたときに一度だけ生成され、決して
外部に送信されません。ファイルが存在しない場合 — 例えば手動で追加された応募の場合 —
パネルは失敗するのではなく、単に監査するものがないことを示します。

## 一見の価値がある理由

フラグの付いた記述は非難ではありません。それは招待です。プロフィールが
あなたが本当に行ったことを単に欠いているだけかもしれません — プロフィールを修正して再実行しましょう。
言い回しが、あなたが安心して擁護できる範囲を超えてしまったのかもしれません — 送信する前に
やわらげるか削除しましょう。いずれにせよ、来歴パネルは *「信じてください、何も捏造していません」* を
1行ずつ検証できるものに変えます。

**関連:** [`/apply`](/ja/docs/commands/apply#every-claim-traced-to-its-source)
は台帳が生成される場所であり、このパネルはそれを読み返す場所です。
