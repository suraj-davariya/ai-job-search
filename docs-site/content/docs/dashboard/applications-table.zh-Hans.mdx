图表用于把握全局，而表格才是你真正干活的地方。每一份申请就是一行，最新的排在最前面，求职过程中的日常——*"招聘方打来电话了，把它移到 Interview"*——都在这里完成。

## 查找内容

每一列都可以排序。筛选器按状态、行业、岗位类型或渠道缩小表格范围，并实时更新。表格上方的[流水线条带](/zh-Hans/docs/dashboard/charts#pipeline-strip)同时充当状态筛选器——点击某个分组，表格随之联动。点击某一行会打开一个抽屉，展示完整记录：每一个字段、你的笔记，以及指向所发送的那份确切 CV 和求职信 PDF 的链接。

## 编辑——恰好只有两项

只有 **status**（状态）和 **notes**（笔记）可以编辑（*last updated*（最后更新）日期会自动打上时间戳）。其他一切——日期、公司、岗位、契合度分数、文件路径——都是 `/apply` 写入的事实记录，仪表盘将其视为只读。

## 状态状态机

状态不是自由文本；它们是七个固定阶段，仪表盘只提供合理的流转选项。试试看——选择一个状态：

<StatusFlowDemo />

完整规则如下：

| 从 | 可流转至 |
|------|------------|
| Draft | Sent, Withdrawn |
| Sent | Interview, Rejected, Withdrawn, Closed |
| Interview | Offer, Rejected, Withdrawn, Closed |
| Offer / Rejected / Withdrawn / Closed | — 终态 |

为什么这么严格？因为这些状态会喂给[面试率 KPI](/zh-Hans/docs/dashboard/kpis) 和[状态图表](/zh-Hans/docs/dashboard/charts)——只有当 "Sent" 始终代表同一含义时，这些数字才值得一读。非法的流转在写入之前就会被拒绝，因此一次误点不会悄悄地破坏你的统计数据。

有两个细节值得了解：

- **终态并不意味着隐藏。** Rejected、Withdrawn 和 Closed 的行会保留在表格中，以灰显方式呈现。你的历史也是数据的一部分——看着 Rejected 这一堆慢慢变成 Interview 那一堆，本身就是一种进步。
- **终态搞错了？** 仪表盘不会把一行从终态移出。你的追踪表是[一个由你掌控的纯文本文件](/zh-Hans/docs/your-data)，所以真正的错误总是可以在任何电子表格应用中修正。

## 手动添加一行

在 CareerForge 之外申请了某个职位——内推、上门拜访或一封邮件？**+ New** 按钮可以手动添加一行，让你的统计数据保持完整。日期、公司和岗位为必填项；CV 和求职信字段则保持为空即可。
