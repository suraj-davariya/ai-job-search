## Problem

v1.2 is the last planned milestone. It has five strands (WBS T-100–T-104), and a fresh
audit shows two of them are much smaller than the plan assumed:

- **Interview prep (T-100)** — `07-interview-prep.md` already ships the STAR framework,
  tough questions, questions-to-ask, follow-up etiquette, **and** roleplay guidelines.
  Only three acceptance-criteria gaps remain (below). This is polish, not a build.
- **`/reset` (T-101)** — still a 4-line stub; needs the full command (REQ-0080–0085).
- **Portal adapter pattern (T-102)** — only the web-search path exists; ADR-0004's
  structured-adapter pattern has no concrete example for contributors.
- **Research agent (T-103)** — `.claude/agents/` has only `reviewer.md`; no research agent.
- **E2E polish (T-104)** — final traceability/consistency pass.

## Goals

1. `/reset` fully implements REQ-0080–0085: scope selection, pre-reset inventory, exact
   `RESET` confirmation, scoped execution that **preserves framework files**, and
   post-reset guidance.
2. Interview prep meets every REQ-3051–3054 acceptance criterion.
3. A working, documented **example portal adapter** exists so a contributor can add a
   market without touching core (ADR-0004), plus a contributor guide.
4. A reusable **research agent** definition exists, usable by `/apply`, `/upskill`, and
   `/expand` for company/market research in an isolated context.
5. A final consistency pass leaves docs, roadmap, and traceability honest.

## Non-goals

- Building real, market-specific portal adapters (the example is a *pattern*, not a
  shipped Indeed/LinkedIn integration — those stay community/optional per ADR-0004).
- Rewriting the interview-prep framework (it already exists — only close the gaps).
- Any change to the dashboard or demo-domain logic.

## Design decisions

### `/reset` — full command, framework-preserving (REQ-0080–0085)

A Profile Manager operation (ARCH-0010, "Profile Resetter"), built as a full command in
the `setup.md`/`expand.md` style. Three scopes: `profile`, `documents`, `all`.

- **Inventory before deletion (REQ-0081):** show, per file/folder, whether it has content
  or is already empty, and explicitly list what is **never** touched.
- **Hard confirmation (REQ-0082):** the user must type exactly `RESET`; anything else
  cancels with "nothing was changed".
- **Framework preservation (REQ-0083):** profile/behavioral files revert to blank section
  headers; CV templates keep LaTeX structure (only profile statements cleared); interview
  prep keeps the framework (only STAR examples cleared). `03-writing-style.md`,
  `04-job-evaluation.md` framework, and `06-cover-letter-templates.md` are **never**
  touched. This is the mirror of what `/setup` and `/expand` write.
- **Documents reset (REQ-0084):** delete user files in `documents/*` subfolders
  (applications recursively) but preserve folder structure and `documents/README.md`.
- **Guidance (REQ-0085):** list what was actually cleared vs already empty, and suggest
  `/setup` next.

`/reset` is destructive, so it leans hardest on the human-in-the-loop and explicit-
confirmation invariants. It is **not** wired into the dashboard console allowlist (a
destructive command shouldn't be one-click from a browser) — confirmed: no `reset` entry.

### Interview prep — close the three gaps (REQ-3051/3052/3054)

The framework is complete; add only:
- REQ-3051: a **"You don't have X"** tough-question entry (handling a missing requirement
  honestly — the minimum set names it explicitly).
- REQ-3052: bring each "Questions to Ask" category to **3–5** questions (currently 2).
- REQ-3054: add the **"don't call to 'stand out'"** guidance and the "brief follow-up
  acceptable after 2+ weeks with no stated timeline" nuance.

These are additive edits to a framework file; `/reset` preserves them (only STAR examples
clear).

### Portal adapter pattern — example at `.agents/skills/` (ADR-0004)

ADR-0004 specifies adapters at `.agents/skills/<portal>/` with `SKILL.md`, `cli/` (Bun
TypeScript), and optional `url-reference.md`. `.agents/` is deliberately **not** the
Claude-Code auto-load path (`.claude/skills/`), so an example there documents the pattern
without being activated as a broken live skill — and it isn't gitignored (only
`.agents/state/` is). 

Deliverable: `.agents/skills/example-portal/` with a documented `SKILL.md`, a runnable
`cli/` skeleton (`package.json` + `index.ts` implementing the input→structured-output
contract and the exit-code contract), and `url-reference.md`. Plus a short contributor
section pointing to it. The example is self-contained and clearly marked "template — copy
me", and falls back to the web-search path conceptually (it doesn't scrape any real site —
it returns clearly-labelled example data so `bun run` works end-to-end as a teaching
tool).

### Research agent — reusable, isolated (T-103)

`.claude/agents/research-agent.md`, modeled on `reviewer.md`: a fresh-context agent that
researches a company, market, or topic via web search and returns **structured, sourced
findings** — never fabricating, always citing URLs. Spawnable by `/apply` (company
angles), `/upskill` (market demand), and `/expand` (enrichment). Content-only; reads
nothing from disk it isn't given; does not edit files.

## Architecture & constraints

- **Prompt-as-code** (ARCH-0001) for `/reset`, the agent, and the adapter `SKILL.md`;
  the adapter `cli/` is real Bun/TypeScript (the one compiled-language touch, per ADR-0004
  and the tech stack). Verification is grep/trace plus a `bun`/`node` smoke run if a
  runtime is present.
- **No fabrication** (ARCH-0007) across `/reset` (never deletes what it didn't list),
  the research agent (cites sources), and the adapter (example data is labelled as such).
- **Destructive-safety** for `/reset`: inventory + exact-string confirmation + explicit
  preserve-list.

## Doc-sync obligations (CLAUDE.md contract)

`/reset` is a new command ⇒ `docs-site/content/docs/commands/reset.mdx` + meta nav;
README `/reset` section, removal from "Planned" table, **roadmap v1.2 → ✅ Complete**,
badges + tree note. Reference the adapter example from the contribution guide. No
dashboard/demo change ⇒ parity test unaffected.

## Risks

| Risk | Mitigation |
|------|------------|
| `/reset` deletes framework/user data unexpectedly | Inventory-first, exact-`RESET` confirmation, explicit never-touch list, "list what was actually cleared vs already empty". |
| Example adapter mistaken for a real, working portal | Clearly labelled "template"; returns labelled example data; lives in `.agents/` (not auto-loaded). |
| Research agent treated as source of truth for candidate claims | Agent returns *research*, not candidate facts; consumers (e.g. `/apply`) independently verify, same as the reviewer's company angles (REQ-2042). |
